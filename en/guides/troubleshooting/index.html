Troubleshooting Guide

Fix Common FAFE Problems

Most failures come from Windows input permissions, a minimized game window, language/template mismatch, or timings that are too fast for the current frame rate.

FAFE simulates keyboard and mouse input. Automation may carry account risk; use it at your own discretion.

Input does nothing

  • If Forza runs as administrator, run FAFE as administrator.
  • Enable automatic English IME switching when using a Chinese input method.
  • Do not minimize the game. Background input supports covered and unfocused windows, but minimized games stop rendering frames.
  • Increase the function?s Post-key Wait or Menu Tap Interval if only some presses are missed.

Detection waits forever

  • This is intentional: FAFE waits until the expected screen appears or you press F9.
  • Confirm the game-menu language matches the selected template language.
  • Keep the relevant UI element inside its normal screen position and avoid covering it with another in-game panel.
  • Enable debug snapshots to see the search region and best match.

Stutter or high CPU usage

  • Leave OCR off unless it is needed to confirm weak text matches.
  • Keep ROI-only detection enabled.
  • Raise detection intervals on weaker hardware.
  • Disable debug snapshots after diagnosis so image files are not written continuously.

Create a useful bug report

  1. Let the automation fail and stay on the exact failed screen.
  2. Press F12 while the game is still on that screen.
  3. Review the generated zip, which contains an annotated detection screenshot, settings, basic system information, and logs.
  4. Attach it to a new post in the Discord bug-report channel.
