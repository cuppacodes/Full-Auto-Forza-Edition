Part 9 — Reporting Bugs

Reporting Bugs

1

One-press diagnostic bundle

Press F12 to generate a diagnostic bundle — an annotated detection screenshot (showing the search region, the match result and where the mouse click landed) and your logs — zipped, and the reports folder opens automatically.

2

When to press it

Let the script run until it fails and stay on the screen where it failed, then press F12 (with the game in focus). This captures the exact screen detection got stuck on. Don't switch to another screen first — stay on the failing one so the problem is visible.

3

Why a hotkey, not a button

The 🐞 Report a Bug button only opens this tutorial — clicking a button would foreground FAFE and screenshot the wrong window. The actual capture must happen via the global F12 hotkey while the game is focused.

4

Privacy

The bundle includes your settings, logs and basic system info (OS, CPU, etc., written into log.txt). Review it before sharing. The system info is only there to check whether a problem is hardware-related (e.g. whether certain CPUs are more prone to it).

5

Rebind the hotkey

Change the report hotkey in Settings (default F12).

✦

How to report

Once the file is generated, open a new post in the Discord bug-report channel below and attach the zip. Please don't post it in general or DM me — posting there keeps a record and lets others reference it.
